# HNG-FrontEnd-stage0 Todo Card

A simple, accessible, and responsive Todo Card built for the HNG Frontend Stage 0 task.  
This project focuses on **semantic HTML, accessibility, responsiveness, and testability using data-testid attributes.**

---

## 🚀 Live Demo
[https://eniola-henry.github.io/HNG-FrontEnd-stage0/]

---

## 📁 Features

- Todo task card UI
- Task title and description
- Priority indicator (Low / Medium / High)
- Status indicator (Pending / In Progress / Done)
- Due date display
- Live “time remaining” updates
- Mark task as complete (checkbox toggle)
- Tags system (work, urgent, design)
- Edit and Delete buttons (UI actions)

---

## 🧪 Testing Requirements

This project includes strict `data-testid` attributes for automated testing:

- `test-todo-card`
- `test-todo-title`
- `test-todo-description`
- `test-todo-priority`
- `test-todo-due-date`
- `test-todo-time-remaining`
- `test-todo-status`
- `test-todo-complete-toggle`
- `test-todo-tags`
- `test-todo-edit-button`
- `test-todo-delete-button`

---

## 🛠️ Tech Stack

- HTML5 (Semantic Markup)
- CSS3 (Responsive Design)
- Vanilla JavaScript (DOM Interaction + Time Logic)

---

## 📱 Responsiveness

- Mobile-first design
- Works from **320px to desktop screens**
- Buttons stack properly on small screens
- No horizontal scrolling

---

## ♿ Accessibility

- Keyboard navigable
- Proper focus states
- ARIA labels for interactive elements
- Semantic HTML (`article`, `time`, `button`, `input`)

---

## ⚙️ How It Works

- Due date is set in JavaScript
- Time remaining updates dynamically every 60 seconds
- Checkbox toggles task completion state
- Status updates automatically based on completion

## 📌 Project Goal

This project was built to demonstrate:

- Ability to follow strict UI specifications
- Clean and maintainable frontend code
- Accessibility awareness
- Accurate implementation of test-driven UI requirements
